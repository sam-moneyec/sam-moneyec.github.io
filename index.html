TEORÍA DE CONJUNTOS

Capítulo 1

1. INTRODUCCIÓN: EL MENÚ

Antes de entrar en la matemática formal y las definiciones abstractas, imaginemos un escenario cotidiano para comprender la naturaleza de esta operación. Supón que vas a un restaurante y ofrecen un "Menú del Día" que te permite elegir entre dos categorías distintas.

Primero, tienes el conjunto de Platos Principales, donde las opciones son únicamente Hamburguesa o Pizza. Segundo, tienes el conjunto de Postres, que ofrece Helado, Pastel o Fruta. La pregunta lógica es: ¿Cuántas combinaciones diferentes de menú completo puedes pedir?

Si las listamos una por una (Hamburguesa con Helado, Hamburguesa con Pastel, etc.), descubriremos que hay un total de 6 combinaciones posibles. Sin saberlo, hemos realizado una operación matemática fundamental: hemos calculado el Producto Cartesiano de dos conjuntos. El resultado no es una suma, sino una multiplicación de posibilidades que crea un nuevo conjunto de parejas.

2. DEFINICIÓN TÉCNICA

En matemáticas, específicamente en la teoría de conjuntos, el producto cartesiano es una operación que se realiza entre dos conjuntos para crear un tercero. Recibe su nombre en honor a René Descartes, el creador de la geometría analítica y del famoso plano cartesiano.

La Fórmula Explicada

Dados dos conjuntos, el conjunto A y el conjunto B, el producto cartesiano (denotado como A x B) es el conjunto de todos los pares ordenados (a, b), donde el primer elemento 'a' pertenece obligatoriamente al conjunto A, y el segundo elemento 'b' pertenece al conjunto B.

A x B = { (a, b) | a ∈ A y b ∈ B }

Es vital entender la diferencia entre un pequeño conjunto {a, b} y un par ordenado (a, b). En un conjunto normal, el orden no importa; {1, 2} es lo mismo que {2, 1}. Sin embargo, en el producto cartesiano, el orden es fundamental y estricto: el par (1, 2) NO es lo mismo que el par (2, 1).

VISUALIZACIÓN Y CÓMPUTO

Diagrama de Flechas

Este diagrama ilustra perfectamente el concepto: cada elemento del conjunto A (izquierda) lanza una "flecha" a todos los elementos del conjunto B (derecha).

El total de flechas (3 x 2 = 6) representa los pares ordenados resultantes.

Generación Matricial en MATLAB

% Definimos los conjuntos como vectores
A = [1, 2, 3];
B = [4, 5];

% Usamos ndgrid: la forma eficiente en ingeniería
% para crear las rejillas de combinaciones
[GridA, GridB] = ndgrid(A, B);

% 'Aplanamos' las rejillas en columnas y las unimos
ParesOrdenados = [GridA(:), GridB(:)];

% Mostramos el resultado en consola
disp('--- Pares Ordenados Generados (A x B) ---');
disp(ParesOrdenados);

Nota de Ingeniería: La función ndgrid es fundamental en simulaciones numéricas para generar espacios de coordenadas n-dimensionales de manera vectorizada, evitando bucles lentos.

CASOS DE USO EN INGENIERÍA

Esto no es solo teoría. Es la base de tecnologías que usas a diario:

1. Tu Monitor (Resolución)

Una pantalla Full HD tiene 1920 columnas y 1080 filas. Cada píxel es un par ordenado (x, y). Tu pantalla es el Producto Cartesiano de Columnas x Filas.

2. Bases de Datos (SQL CROSS JOIN)

Si haces SELECT * FROM Usuarios, Pedidos sin condición, la base de datos colapsará al intentar crear un Producto Cartesiano masivo de todos contra todos.

3. Ataques de Fuerza Bruta

Un hacker que intenta adivinar una contraseña está recorriendo el Producto Cartesiano de todos los caracteres posibles para encontrar la combinación correcta.

Lógica Equivalente en MATLAB
% Código para generar combinaciones como el simulador

% 1. Definir los conjuntos (Arrays o Celdas de texto)
SetA_Datos = {'S', 'M', 'L'};
SetB_Datos = {'Rojo', 'Azul'};

% 2. Bucles anidados para recorrer todo (Lógica clásica)
disp('Generando combinaciones...');
contador = 1;
for i = 1:length(SetA_Datos)
    for j = 1:length(SetB_Datos)
        % Crear el par
        elementoA = SetA_Datos{i};
        elementoB = SetB_Datos{j};
        
        % Mostrar en formato de par ordenado
        fprintf('Par %d: (%s, %s)\n', contador, elementoA, elementoB);
        contador = contador + 1;
    end
end
fprintf('Total generado: %d pares.\n', contador-1);

VIOLACIÓN DE UNICIDAD

Observa atentamente el elemento 2 en el conjunto de partida (A).

Este elemento tiene DOS flechas salientes: una apunta al elemento 'y' y otra apunta al elemento 'z'.

VEREDICTO: NO ES UNA FUNCIÓN

Para que sea una función válida, a cada elemento del dominio le debe corresponder exactamente una única imagen en el codominio. El elemento 2 rompe esta regla fundamental.

CLASIFICACIÓN

Slides 9-10

Inyectiva

Cada elemento de B tiene a lo sumo una anti-imagen. (1 a 1).

Sobreyectiva

El Rango es igual al Codominio. No sobran elementos en B.

Biyectiva

Es inyectiva y sobreyectiva simultáneamente. Perfecta.

Validación Inyectividad
n = 1:10; 
y = 2*n + 1; 
if length(unique(y)) == length(y)
    disp('Es Inyectiva: No hay colisiones');
end
